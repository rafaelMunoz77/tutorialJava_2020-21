Corrigiendo los fallos de sonido

(c) Alexander Hristov

Si hemos probado nuestro el programa anterior, nos habremos dado cuenta de dos cosas : en primer lugar el fps ha bajado considerablemente y - lo que es mas grave - el juego ha comenzado a funcionar "a trompicones". El problema es que Java necesita preparar un sonido para su reproducción y este proceso lleva tiempo. Cada vez que solicitamos que se toque algo, como lo estamos haciendo desde nuestro bucle principal, el programa se detiene mientras Java perpara el sonido. Un planteamiento mejor - que además arregla el problema de los trompicones - es crear un nuevo hilo temporal que y desde este nuevo hilo solicitar la ejecución de un sonido. De esta forma el programa principal puede continuar sin que tengamos que esperar. Afortunadamente, tenemos todas las peticiones de reproducción de sonidos centralizadas en la clase SoundCache de forma que será fácil añadir el cambio sin afectar al resto del programa:
